## 一、结局判定时机

```
12月底触发 monthly_m12_judgment 事件后进入结局判定：

优先级（从高到低）：
  1. 隐藏结局（条件苛刻，最先检查）
  2. 失败结局（已在游戏中途触发过的）
  3. 胜利结局（按满意度和质量排序）
  4. 兜底：外卖骑手
```

## 五、添加新结局

```markdown
### 给 AI 的 Prompt

请阅读 `_schema.md` 和 `endings.md`。

请设计 3 个新的隐藏结局：
- 每个结局需要苛刻的触发条件
- 需要完整的文学性结局文本（3~5段）
- 标明奖励金币（隐藏结局通常 50~130）
- 条件应该覆盖不同的游戏风格（省钱流/社交流/技术流等）
```
