## 一、月份阶段

| 阶段 | 月份 | 名称 | 特点 |
|:---|:---|:---|:---|
| 🌱 | 1月 | 新手上路 | 只有豆包，任务简单，教学月 |
| 📖 | 2–3月 | 摸索期 | 群雄并起，模型涌现 |
| ⚔️ | 4–6月 | 生存战 | 裁员开始，Deadline压力 |
| 🔥 | 7–9月 | 冲刺期 | 半年考核，最后机会 |
| 🏆 | 10–12月 | 审判日 | 最终结局揭晓 |

## 二、每月流程

```
月初：
  1. 发工资 (+salary)
  2. 扣生活费 (-5000)
  3. 自然恢复（brain +10, hp +5）
  4. 模型解锁事件（如果有）
  5. 月份大事件

工作日（3~5天）：
  每天选模型写代码 → 可能出Bug → 可能熬夜
  可能穿插随机事件 / 选择事件

月末结算：
  1. 本月代码质量统计
  2. 老板满意度变化
  3. 同事关系变化
  4. 可选：购买Token / 切换模型
  5. 下月预告
```

---

## 三、事件生成约束

1. **事件ID格式**：`monthly_m{N}_{desc}_{seq}`（N为月份数字）
2. **system 字段**：`"monthly"`
3. **大事件必须 `once: true`**
4. **每月应有1个大事件** + 可选1-3个子事件
5. **月份递进**：事件压力应随月份递增
6. **关键月份**：4月(第一轮裁员)、8月(半年考核)、12月(年终审判) 必须有重大事件
